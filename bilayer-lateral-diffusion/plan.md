# Bilayer Lateral Diffusion Workflow

## Project Goal
- Create a new root-level workflow that measures DOPC bilayer lateral diffusion across damping, mass, and temperature conditions.
- Make the workflow runnable locally and on Slurm with array parallelism.
- Keep the implementation self-contained under `bilayer-lateral-diffusion/`.

## Architecture & Key Decisions
- Reuse the existing bilayer preparation and stage-conversion path in `py/martini_prepare_system.py`.
- Keep v1 bilayer-only so global thermostat-timescale and mass scaling are enough to scan fluidity without engine changes.
- Measure fluidity from leaflet-resolved DOPC `PO4` lateral MSD on the production stage `.up` trajectory directly, with XY unwrapping and bilayer COM drift removal.
- Use the bundled DOPC template size as a relative screen, not an absolute diffusion benchmark.
- Follow the existing `SC-training/` Slurm-array staging pattern and the `ConDiv_symlay` path-resolution pattern.
- For post-processing after stage-6 cleanup, analyze surviving `stage_7.0.up` files directly and let the analyzer fall back to the stage-7 file as its own leaflet reference when no stage-6 checkpoint exists.
- Write post hoc analysis outputs under a separate `stage7-analysis/` tree so they do not overwrite the original simulation-time results.
- Generate a separate download archive that can include both post-analysis outputs and the surviving `stage_7.0.up` files.
- For downloaded `stage7-analysis/` trees, validate completeness from the manifest plus assembled outputs before generating any plots or recommendations.
- Use the user-provided Upside calibration of `40 ps` per integration step with the workflow production step size of `0.01`, so `1` workflow time unit corresponds to `4 ns`.
- For temperature-specific calibration, keep each scanned Upside temperature fixed and convert it to Kelvin using `1.0 T_up = 350.588235 K` from `AGENTS.md`.
- Report converted diffusion as:
  - `nm^2/ns = diffusion_mean_nm2_per_time / 4`
  - `um^2/s = diffusion_mean_nm2_per_time * 250`
- Rank candidate `(temperature, damping, mass_scale)` settings from the converted diffusion values and fit quality, while treating the final target diffusion as a calibration choice rather than a hard-coded constant.
- Use the DOPC supported-bilayer diffusion data from Bag, Yap, and Wohland (BBA 2014, Table 1) as the explicit physical target curve:
  - `D_298K = 2.46 um^2/s`
  - `D_303K = 2.77 um^2/s`
  - `D_308K = 3.07 um^2/s`
  - `D_313K = 3.40 um^2/s`
  - `E_Arr = 17.66 kJ/mol`
- Derive per-temperature bilayer corrections by matching the scan to that Arrhenius target at fixed temperature, and prefer a stable damping window over noisy high-`tau` outliers if damping has only a weak effect.
- The thermostat implementation must be checked against the Einstein relation before trusting any mass/damping calibration. If the thermostat noise is not mass-aware, previous mass/damping recommendations are invalid and must be treated as provisional only.
- After the thermostat fix, the refreshed `stage7-analysis/` download must be revalidated and all report artifacts regenerated before any hybrid timescale recommendation is reused.
- Because workflow `16` should now follow the same thermostat default as the standard Upside examples, the next bilayer scan should treat `tau = 5.0` as the reference damping and explore deeper mass scaling before reopening damping as a free knob.
- The next recommended rerun is a fixed-temperature, fixed-damping mass-only scan:
  - `damping = 5.0`
  - `temperature = 0.70, 0.80, 0.90, 1.00, 1.10`
  - `mass_scale = 0.1, 0.05, 0.02, 0.01, 0.005, 0.002`
  - `replicates = 3`
  - keep the existing MD stage lengths unless stability forces a shorter pilot pass.
- Stage-7 post-analysis wrappers must follow the selected run preset as well, so the focused `tau5_mass_only` rerun can be analyzed without manually overriding `BILAYER_DIFF_BASE_DIR`.
- The next joint mass+damping follow-up should keep temperature fixed to the same Upside temperature and probe a moderate damping window around the standard example baseline:
  - `damping = 5, 8, 12, 16, 24`
  - `temperature = 0.70, 0.80, 0.90, 1.00, 1.10`
  - `mass_scale = 0.02, 0.01, 0.005, 0.002, 0.001`
  - `replicates = 3`
  - rationale:
    - `tau = 5` is the aligned Upside baseline,
    - `8 -> 24` probes weaker damping without jumping immediately into the previously noisy extreme-`tau` regime,
    - `0.02 -> 0.001` focuses around the best current low-mass region while pushing one decade lower than the last robust hot-temperature points.
- The next boundary-focused rerun should target the transition between the robust `tau = 5-8` regime and the unstable `tau = 16-24` regime instead of repeating the full coarse joint grid:
  - `damping = 5, 8, 10, 12, 14, 16`
  - `temperature = 0.70, 0.80, 0.90, 1.00, 1.10`
  - `mass_scale = 0.02, 0.015, 0.01, 0.0075, 0.005, 0.003, 0.002`
  - `replicates = 3`
  - rationale:
    - keep `tau = 5` and `8` as the robust anchors already observed in the data,
    - fill in `10, 12, 14, 16` to locate where the instability frontier starts,
    - drop `tau = 24` because it is already clearly on the pathological side of the boundary,
    - drop `mass = 0.001` because it repeatedly produced NaNs and exploded diffusion,
    - refine the useful mass window between `0.015` and `0.002` where the speed / stability tradeoff is actually happening.
- The next numerical-stability diagnostic should hold the same boundary-point scientific grid and vary only the time integration surface:
  - `integrator = v, nvtc`
  - `dt = 0.01, 0.0075, 0.005`
  - `damping = 5, 12, 14, 16`
  - `temperature = 0.70, 0.90, 1.10`
  - `mass_scale = 0.005, 0.003, 0.002`
  - `replicates = 3`
  - rationale:
    - compare the current production integrator `v` against `nvtc` on the same unstable dryMARTINI boundary points,
    - shrink `dt` before concluding the instability is intrinsic to the force model,
    - keep `nvtc` at `max-force = 0.0` by default so the comparison measures the stepper first rather than force clipping,
    - stage each `(integrator, dt)` pair under its own run directory to keep manifests and stage-7 post-analysis simple.

## Known Errors / Blockers
- The bundled DOPC system is small (`72` lipids), so the reported diffusion coefficients are for relative ranking across conditions.
- Absolute physical calibration will likely need a larger tiled bilayer in a later pass.
- The reduced smoke run used very short MD stages and exhibited nonphysical energy growth, so it validates workflow plumbing only, not bilayer physics under production settings.
- The available experimental DOPC target curve is only directly constrained near `298-313 K`; recommendations outside that range are extrapolations and should be treated as such.
- If the thermostat is missing mass-dependent noise, the existing bilayer mass/damping calibration outputs are not physically trustworthy and must be regenerated after the fix.
- Even after aligning workflow `16` to the standard example thermostat default, its smaller hybrid timestep means the borrowed `40 ps/step` calibration remains provisional for stage `7.0`.
